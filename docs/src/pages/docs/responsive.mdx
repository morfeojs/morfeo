# Responsive

Handling responsiveness in Morfeo it's straightforward:

```tsx filename="components/Card" showLineNumbers {5-8}
import { morfeo } from '../morfeo';

const classes = morfeo.css({
  container: {
    px: {
      default: 'm',
      sm: 's',
    },
    bg: 'white',
    color: 'text',
  },
});

export function Card() {
  return (
    <div className={classes('container')}>
      This card will have different padding horizontal based on the screen size!
    </div>
  );
}
```

## How it works

Instead of plain values/theme-values you can pass to any style property an object called `Responsive Object` that has the following shape:

```typescript showLineNumbers
type ResponsiveObject<V> = {
  default?: V;
  // keys of the breakpoints slice
  xs?: V;
  sm?: V;
  md?: V;
  lg?: V;
};
```

I don't think we need to specify what `default` does, instead, values passed to `xs`, `sm`, `md` and `lg` are applied only to specific screen sizes, in particular the ones specified into the `breakpoints` theme's slice, these are the defaults:

```typescript showLineNumbers
const theme = {
  // ...
  breakpoints: {
    lg: '1920px',
    md: '1366px',
    sm: '768px',
    xs: '375px',
  },
};
```

### Mobile first

The generated media queries are, by default, mobile-first:

```css showLineNumbers
@media screen and (min-width: 375px) {
  /*  */
}
@media screen and (min-width: 768px) {
  /*  */
}
@media screen and (min-width: 1366px) {
  /*  */
}
@media screen and (min-width: 1920px) {
  /*  */
}
```

## Customizations

### Custom breakpoints

You can customize both the keys and the values of the breakpoints slice with a custom theme:

```typescript showLineNumbers
const theme = {
  // ...
  breakpoints: {
    large: '1920px',
    medium: '1366px',
    small: '768px',
    xsmall: '375px',
  },
};
```

In this case, also the `Responsive Object` is adapted to **your new language**:

```typescript showLineNumbers
morfeo.css({
  container: {
    padding: {
      default: 'm',
      large: 'l',
      medium: 'm',
      small: 's',
      xsmall: 's',
    },
  },
});
```

### Custom media queries

Other than the `breakpoints` slice, you can also customize how the media queries are generated by using the `mediaQueries` theme's slice, for example:

```typescript
const theme = {
  // ...
  mediaQueries: {
    xs: '@media screen and (min-width: {{xs}})',
    sm: '@media screen and (min-width: {{xs}}) and (max-width: {{md}})',
    md: '@media screen and (min-width: {{md}}) and (max-width: {{lg}})',
    lg: '@media screen (min-width: {{lg}})',
  },
};
```

The values wrapped with `{{value}}` are placeholders that will be overridden by the values of the breakpoints when the media queries are generated.
